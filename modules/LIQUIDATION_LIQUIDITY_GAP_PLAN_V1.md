# 清算热图 / 流动性热区能力补强计划 V1

更新时间：2026-03-14

## 一、目标
补强 ZeroOne 对 BTC 及高波动资产的拐点预警能力，重点解决：
- 看得出方向，但抓不住瀑布触发位
- 能做结构判断，但缺少清算/流动性触发验证
- 对短时大幅回撤的提前预警能力不足

## 二、当前缺口
### 1. 专业级清算热图缺失
- 当前仍无标准 liquidation heatmap 数据源
- 无法直接看到大规模清算堆积区

### 2. 流动性热区识别不够强
- 当前只能通过 book / 价格结构 / funding / OI 做替代判断
- 对“哪一带一旦破位就会引发连锁踩踏”识别不够稳定

### 3. 短周期拐点信号不足
- 缺少将：
  - 关键价位
  - OI 变化速度
  - funding 变化速度
  - 盘口抽单
  - 短时成交量异常
统一拼成预警链的能力

## 三、当前已有基础
### 已有模块
- `modules/LIQUIDITY_HEATMAP_ALT_V1.md`

### 当前可用能力
- `hyperliquid`
- `funding-rate-scanner`
- Binance depth / 公开行情
- 价格结构分析

## 四、补强目标拆解
### 第一层：替代版增强
先把现有替代方案升级得更接近“实战可预警”：
1. 标记关键上方/下方流动性密集区
2. 标记潜在挤空带/挤多带
3. 标记一旦跌破/突破容易加速的区间
4. 加入短周期 OI / funding / volume 联动判断

### 第二层：预警规则化
形成固定预警规则，例如：
- 价格跌破关键承接位 + OI 快速回落 + funding 未明显转弱 → 多头踩踏风险
- 价格上冲关键位 + funding 仍负 + OI 抬升 → 挤空延续风险

### 第三层：专业数据源升级（后续）
后续若接入更强数据源（如 Coinglass 同类），再把替代方案升级为专业版。

## 五、执行优先级
### P1
升级现有 `LIQUIDITY_HEATMAP_ALT_V1`，让它从“替代判断模块”变成“拐点预警辅助模块”。

### P2
补一份《短周期拐点预警规则 V1》：
- 价格
- OI
- funding
- volume
- book
- 失效条件

### P3
评估专业热图数据源接入路径。

## 六、当前结论
这一步不是重新推翻现有模块，而是：
- 先承认 `LIQUIDITY_HEATMAP_ALT_V1` 还不够
- 再围绕它继续补强
- 最终把它升级成 ZeroOne 的“高波动资产拐点预警能力”之一
