# 链上 / 鲸鱼 / 现货资金验证模块 V1

更新时间：2026-03-14

## 一、目标
建立一套用于验证市场走势真实性的资金验证层，重点解决：
- 当前上涨/下跌到底是合约驱动，还是现货驱动
- 是否存在鲸鱼提前行动
- 链上资金是否支持当前行情延续
- 是否只是表面反弹/回撤，而没有真实资金接力

## 二、核心问题
1. 大额地址是否出现新增异动？
2. 交易所净流入 / 净流出是否异常？
3. 稳定币是否有增量流入？
4. 鲸鱼是否在关键位提前布局或撤退？
5. 当前行情是否得到现货资金支持？

## 三、分析对象
### A. 链上资金
- BTC / ETH 相关链上转账
- 交易所相关钱包流向
- 稳定币增减（如可得）

### B. 鲸鱼行为
- 大额地址异动
- 是否有集中转所 / 提币 / 增持 / 减持
- 是否在关键价格区间提前动作

### C. 现货与合约区分
- 若价格变化明显，但链上和鲸鱼侧无支持，更可能是合约驱动
- 若价格变化伴随现货流入 / 鲸鱼增持，更可能是可持续性更强的走势

## 四、当前可用能力
1. `crypto-whale-monitor`
- 用于监控重点鲸鱼钱包异动

2. `etherscan`
- 用于 EVM 链上基础查询

3. 市场结构侧能力
- `hyperliquid`
- `funding-rate-scanner`
用于对照判断：当前更偏现货还是偏合约驱动

## 五、输出模板
1. 当前价格变化
2. 链上资金变化
3. 鲸鱼行为变化
4. 交易所流向判断
5. 稳定币增量判断（如可得）
6. 当前更偏：
- 现货驱动
- 合约驱动
- 鲸鱼推动
- 无明显链上确认
7. 对行情延续性的含义
8. 风险提示

## 六、使用原则
1. 不把单一鲸鱼动作直接等同于趋势
2. 不把链上“未见证据”误判成“绝对没有资金动作”
3. 必须把链上/鲸鱼侧与价格、OI、funding 结合起来看
4. 重点用于验证行情真实性，而不是单独预测方向

## 七、当前结论
该模块将作为 ZeroOne 在高波动资产分析中的关键验证层使用，尤其适用于：
- BTC 大涨/大跌后的真实性验证
- 高波动资产是否有现货接力的判断
- 识别“只是合约拉盘”还是“真实资金进场”
