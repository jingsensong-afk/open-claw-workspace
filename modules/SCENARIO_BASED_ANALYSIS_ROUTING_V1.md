# 场景化分析模板分流机制 V1

更新时间：2026-03-14

## 一、目标
建立一套统一的任务分流机制，确保面对不同类型的分析任务时：
- 先识别任务场景
- 再调用对应模板 / 模块 / 工作流
- 避免模板混用、结构失稳、该用的模块没调出来

## 二、要解决的问题
1. 不同任务适用的分析结构不同
2. 若不先分流，容易混用模板或遗漏模块
3. 模块越来越多后，更需要先判断任务属于哪一种场景

## 三、场景分类（V1）
### A. 实盘异动诊断
适用于：
- 为什么突然涨/跌
- 现在谁在主导
- 会不会延续

默认调用：
- 实盘分析优先机制
- 跨市场联动监控模块
- 新闻 / 宏观触发响应模块
- 短周期拐点预警规则
- 失效条件优先机制

### B. 热点事件投研
适用于：
- 某条新闻 / 事件 / 话题的投研分析
- 资金会流向哪里
- 哪些市场和标的会受影响

默认调用：
- 投研交易框架
- 事件分析模板
- 热点事件映射池模板
- 产业瓶颈 / 价格错配模块
- 市场是否相信验证链

### C. 交易机会筛选
适用于：
- 当前有哪些可交易机会
- 哪些是主交易 / 高弹性 / 观察标的
- 触发/失效条件是什么

默认调用：
- 日内/波段交易判断框架
- 多层信号优先级与汇总机制
- 置信度分级与结论表达机制
- 时间维度分层机制

### D. 复盘纠错
适用于：
- 为什么判断错了
- 漏了什么能力
- 下次怎么补强

默认调用：
- 失效条件优先机制
- 能力缺口分析
- 模块补强计划

## 四、使用原则
1. 先判断任务场景，再回答
2. 同一任务可叠加多个场景，但必须先确定主场景
3. 不能把“实盘异动诊断”和“热点事件投研”混为一谈
4. 不能把“复盘纠错”写成“实时交易建议”

## 五、当前结论
该机制将作为 ZeroOne 的分析调度层之一使用。

作用是：
- 先做任务分流
- 再挂对应模板与模块
- 最终让输出结构更稳定，调用更自动化
