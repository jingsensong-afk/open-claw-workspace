# 清算热图 / 流动性热区替代分析模块 V1（拐点预警辅助版）

## 目标
在未接入 Coinglass / 专用 liquidation heatmap 数据源前，用现有可用能力构建可实战使用的替代分析模块，重点服务于：
1. 识别短时清算带
2. 识别流动性真空区
3. 辅助判断价格一旦突破/跌破后是否容易加速
4. 提升 BTC 与高波动资产的拐点预警能力

## 当前使用能力
1. `hyperliquid`
- 用于读取：quote / funding / OI / L2 book / volume

2. `funding-rate-scanner`
- 用于读取：Funding 方向、极端拥挤币种、Funding 变化

3. 交易所盘口深度（Binance / Hyperliquid）
- 用于读取：买卖盘挂单密集区、短线流动性堆积区、盘口承接抽离

4. 价格结构
- 用于读取：关键高低点、前高前低、趋势成本带、关键止损位

## 分析逻辑
### A. 拥挤方向
- Funding 显著为负：空头付费给多头，说明空头更拥挤
- Funding 显著为正：多头付费给空头，说明多头更拥挤

### B. 杠杆与流动性集中区
- 结合 OI 与价格位置，判断是否存在高杠杆拥挤区
- 结合盘口深度，识别短线容易被扫的流动性密集区
- 结合 volume，看突破/跌破是否有加速基础

### C. 潜在清算带（替代判断）
#### 向上潜在清算带
- 价格上方的关键前高
- 卖盘薄弱区后的挂单真空带
- 空头拥挤时更容易被向上挤压

#### 向下潜在清算带
- 价格下方的关键前低
- 买盘薄弱区后的挂单真空带
- 多头拥挤时更容易被向下清算

### D. 拐点预警辅助逻辑
#### 向上加速预警
若同时出现：
1. 价格逼近关键前高 / 关键压力位
2. Funding 仍负或未明显翻正
3. OI 抬升
4. 上方挂单真空明显
则说明：
- 容易触发挤空延续
- 突破后价格可能快速加速

#### 向下加速预警
若同时出现：
1. 价格跌破关键承接位 / 前低
2. OI 快速回落或高位钝化后转降
3. 下方买盘承接变薄 / 盘口抽单
4. volume 放大
则说明：
- 容易触发多头止损 / 连锁踩踏
- 下方流动性测试概率提升

## 输出模板
1. 当前价格
2. Funding 方向
3. OI / 24h volume
4. 上方流动性密集区
5. 下方流动性密集区
6. 潜在向上清算带
7. 潜在向下清算带
8. 当前判断（偏挤空 / 偏挤多 / 中性）
9. 拐点预警信号
10. 失效条件
11. 风险提示

## 预警规则（V1）
### A. 挤空延续预警
- Funding 仍负
- OI 抬升
- 价格逼近关键压力位
- 上方挂单真空明显

### B. 多头踩踏预警
- 跌破关键承接位
- OI 快速回落
- 下方盘口抽单
- 成交量显著放大

### C. 假突破 / 假跌破识别
- 价格越过关键位，但 volume 未跟上
- Funding / OI 未形成共振
- 很快回到关键位内侧

- Funding 为负，说明空头一侧更拥挤
- OI 仍高，说明杠杆活跃
- 近价位下方存在较明显买盘承接
- 在无专用 heatmap 的情况下，可初步判断：
  - 短线更偏“向上挤空”的结构
  - 但若价格跌破近端承接位，则会转向下方流动性测试
- 这意味着：
  - 既可做结构判断
  - 也可作为拐点预警辅助使用

## 当前限制
- 不是标准 liquidation heatmap
- 无法直接看到大规模清算聚集区图像化分布
- 对极短时大幅波动的预警能力仍弱于专业热图数据源
- 当前更适合作为“V1 拐点预警辅助模块”，而非完全替代专业清算图
