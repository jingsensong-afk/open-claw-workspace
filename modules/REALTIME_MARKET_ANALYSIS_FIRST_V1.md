# 实盘分析优先机制 V1

更新时间：2026-03-14

## 一、目标
建立一套统一的实盘分析输出顺序，确保面对市场异动、价格波动、行情判断类问题时：
- 先做实盘诊断
- 再给结构判断
- 最后才补原理解释

## 二、要解决的问题
1. 分析容易先讲原理，而不是先看盘面
2. 输出有时更像框架解释，而不是交易级诊断
3. 缺少“先拉数据、先看联动、先下结论”的强制顺序

## 三、核心原则
### 1. 先盘面，后原理
面对实盘问题，默认顺序是：
1. 当前价格/涨跌幅/量能/结构
2. 联动资产状态
3. 当前主导变量
4. 当前结论
5. 最后再补原理解释

### 2. 先结论，后展开
- 先说当前判断
- 再说证据
- 最后说原因

### 3. 原理解释不能替代实盘诊断
- 原理只能补充，不是主回答
- 若没有实盘变量，分析不算完成

## 四、适用范围
适用于：
- BTC / ETH / SOL 等高波动资产分析
- 黄金 / 原油 / 美元 / 美债联动分析
- 美股 / 港股 / 商品盘面判断
- 任何“为什么现在涨/跌”“接下来怎么看”的问题

## 五、固定输出顺序
每次实盘分析默认按以下顺序：
1. 当前盘面快照
2. 当前联动关系
3. 当前主导变量
4. 当前结论
5. 触发条件
6. 失效条件
7. 风险提示
8. 若有必要，再补原理解释

## 六、错误顺序与正确顺序
### 错误顺序
- 先讲理论
- 再推演可能性
- 最后才补数据

### 正确顺序
1. 先拉当前数据
2. 先判断当前模式
3. 再给结论
4. 最后再解释逻辑

## 七、当前结论
从现在开始，ZeroOne 的实盘问答必须优先遵守：
**实盘分析优先机制**

这将用于减少：
- 原理分析过多
- 贴盘不够
- 交易帮助不足
并提升对实时盘面的诊断能力与可执行性。
