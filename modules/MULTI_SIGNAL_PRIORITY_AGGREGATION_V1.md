# 多层信号优先级与汇总机制 V1

更新时间：2026-03-14

## 一、目标
建立一套统一的信号仲裁与汇总机制，解决多个分析模块同时给出不同信号时的裁决问题，确保：
- 信息越来越多时，不会各说各话
- 多层信号能收敛成一个可执行判断
- 不同模块之间有优先级，而不是平铺堆叠

## 二、要解决的问题
1. 技术面、跨市场、链上、新闻、情绪等信号可能互相冲突
2. 若没有优先级，最终判断容易摇摆
3. 模块越来越多，但缺少统一裁决层

## 三、核心原则
### 1. 不是所有信号权重相同
不同模块在不同场景下权重不同。

### 2. 先问“谁在主导”
不是机械叠加信号，而是先判断当前主导盘面的变量是谁。

### 3. 先强信号，后弱信号
- 强信号：价格结构失效、跨市场共振、链上/现货确认、重大新闻触发
- 弱信号：单一情绪指标、单一 KOL 观点、单点噪音数据

## 四、默认优先级（V1）
### 第一优先级
1. 价格结构失效 / 关键位破坏
2. 跨市场联动主导信号
3. 重大新闻 / 宏观触发

### 第二优先级
4. 链上 / 鲸鱼 / 现货资金验证
5. 清算带 / 流动性热区 / 短周期拐点预警

### 第三优先级
6. 情绪辅助信号
7. 单一热点噪音 / 个别舆论脉冲

## 五、使用规则
### A. 若价格结构失效
- 优先改判
- 其他利多解释降级处理

### B. 若跨市场联动与单一资产方向冲突
- 优先判断是否是外部主导变量压制/推动

### C. 若新闻利多，但链上/现货不确认
- 降级为“消息驱动但确认不足”
- 不直接上升到高置信判断

### D. 若情绪很强，但价格/OI/联动不确认
- 只作为辅助，不单独定性

## 六、输出模板
每次汇总时至少给出：
1. 当前主导变量
2. 最强信号
3. 次强信号
4. 被降级处理的信号
5. 最终结论
6. 当前置信度：
- 高
- 中
- 低

## 七、当前结论
该机制将作为 ZeroOne 的“信号仲裁层”使用，负责把：
- 技术面
- 联动面
- 链上面
- 新闻面
- 情绪面
统一收敛成最终判断。

没有这层，模块越多越容易分散；
有了这层，模块越多越容易形成系统优势。
