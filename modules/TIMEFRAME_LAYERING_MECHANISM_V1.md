# 时间维度分层机制 V1

更新时间：2026-03-14

## 一、目标
建立一套统一的时间维度分层机制，确保分析输出时：
- 结论明确属于哪个时间层
- 不同周期的判断不会混在一起
- 避免把短线信号误判成中线趋势，或把中线观点错误用于日内交易

## 二、要解决的问题
1. 不同时间框架下，同一资产结论可能完全不同
2. 若不明确时间层，判断容易打架
3. 周期错配会直接影响执行质量

## 三、时间层定义
### A. 短线
- 5m / 15m / 1h
- 用于：拐点、急拉急跌、盘中结构、触发位

### B. 日内
- 4h / 当日
- 用于：当天主方向、回踩/突破有效性、日内节奏判断

### C. 波段
- 日线 / 3日 / 周线前段
- 用于：几天到一两周的趋势判断、主题延续性

### D. 中线
- 周线 / 月线
- 用于：中周期主趋势、产业逻辑与资产重估

## 四、输出要求
每一份分析至少回答：
1. 当前判断属于哪个时间层？
2. 在更短周期里是否成立？
3. 在更长周期里是否成立？
4. 是否存在“短线反弹，但中线仍弱”的情况？

## 五、使用原则
### 1. 不跨周期偷换结论
- 短线看多 ≠ 中线看多
- 中线看空 ≠ 日内不能反弹交易

### 2. 先定周期，再下判断
任何结论先说明：
- 是短线结论
- 还是日内结论
- 还是波段/中线结论

### 3. 周期冲突时要明确指出
例如：
- 短线偏强
- 日内中性
- 波段仍弱

而不是给出模糊的“看多/看空”。

## 六、输出模板
1. 短线判断
2. 日内判断
3. 波段判断
4. 中线判断
5. 当前主要交易层
6. 风险提示

## 七、当前结论
该机制将作为 ZeroOne 的分析框架底层规则之一使用，帮助减少：
- 周期错配
- 方向正确但时间错误
- 短线/中线结论混淆

这样能让分析更适合直接用于交易执行。
