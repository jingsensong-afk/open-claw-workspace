# Token / 地址 / 热门信息一体化能力模块 V1

更新时间：2026-03-14

## 一、目标
把分散的 token 审计、地址分析、热门 token 信息整合成一体化能力入口，解决：
- 现在能力分散可用，但不够统一
- 同一任务需要切多个模块，输出口径不一致
- 难以快速给出“是否可交易、风险等级、资金行为、热度状态”一体化结论

## 二、整合范围
### A. Token 安全与审计
- `is-token-safe`
- 用于识别 scam / rug / 高风险 token

### B. 地址与链上行为
- `etherscan`
- `crypto-whale-monitor`
- 用于查看地址行为、资金迁移、鲸鱼动作

### C. 热门 token 信息
- 市场价格/涨跌/量能
- 叙事热度与话题扩散（结合 opennews/opentwitter）

## 三、一体化输出目标
每次输出统一给出：
1. Token 基础信息
2. 安全风险等级
3. 地址/鲸鱼行为摘要
4. 热度状态（是否热点）
5. 价格与量能状态
6. 当前判断：
- 可交易
- 仅观察
- 高风险回避
7. 触发条件
8. 失效条件
9. 风险提示

## 四、判断分级
### A. 可交易
- 安全风险可控
- 地址行为无明显异常出货
- 热度与量能匹配

### B. 仅观察
- 风险不明或证据不足
- 热度存在但资金确认不足

### C. 高风险回避
- 审计/安全风险高
- 地址行为异常（疑似出货/诱导）
- 热度与价格结构明显背离

## 五、使用原则
1. 不因“热门”直接等同于“可交易”
2. 不因“涨幅大”忽略安全风险
3. 地址行为与安全评级优先于短时情绪
4. 一体化输出必须包含失效条件

## 六、与现有模块联动
该模块与以下机制联动：
- `INVALIDATION_FIRST_MECHANISM_V1`
- `CONFIDENCE_GRADING_AND_OUTPUT_V1`
- `MULTI_SIGNAL_PRIORITY_AGGREGATION_V1`
- `TIMEFRAME_LAYERING_MECHANISM_V1`

## 七、当前结论
该模块用于把“token审计 + 地址分析 + 热门信息”从分散能力升级为统一入口，补齐 ZeroOne 在该方向的一体化能力缺口。
