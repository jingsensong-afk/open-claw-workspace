# 置信度分级与结论表达机制 V1

更新时间：2026-03-14

## 一、目标
建立一套统一的结论表达机制，确保分析输出时：
- 不同判断的强弱被明确区分
- 用户可以快速分辨哪些是可执行判断，哪些只是观察线索
- 结论不会因为表达方式而显得“全都一样重”

## 二、要解决的问题
1. 同一份分析中，不同结论证据强弱不同，但表达常常差不多
2. 用户难以快速识别：哪些能直接拿来交易，哪些只能观察
3. 缺少“结论强度”和“行动强度”的对应关系

## 三、置信度分级
### A. 高置信
条件：
- 多层信号一致
- 关键位 / 联动 / 链上 / 触发源至少有两到三层确认
- 失效条件清晰

含义：
- 可执行级判断
- 可给出主交易标的 / 触发条件 / 失效条件

### B. 中置信
条件：
- 有一定证据支持
- 但确认不完整，或存在重要变量未验证

含义：
- 可作为观察级或轻仓试错级判断
- 需要等待进一步确认

### C. 低置信
条件：
- 只有单点信号
- 缺少关键确认
- 更像线索或初步猜测

含义：
- 仅作线索
- 不应表述成交易级结论

## 四、输出规则
### 每份分析至少给出
1. 当前结论
2. 当前置信度（高 / 中 / 低）
3. 对应动作：
- 高置信 = 可执行
- 中置信 = 观察 / 小仓位试错
- 低置信 = 仅线索，不做主结论

## 五、表达要求
### 错误表达
- 把低置信判断写得像高置信结论
- 把线索写成机会

### 正确表达
- 明确说明：
  - 当前判断是什么
  - 置信度在哪一档
  - 对应能做什么，不能做什么

## 六、和交易动作的对应关系
### 高置信
- 主交易标的
- 明确触发条件
- 明确失效条件

### 中置信
- 观察标的
- 等待进一步确认
- 可轻仓试错

### 低置信
- 只作为研究线索
- 不升格为正式交易结论

## 七、当前结论
该机制将作为 ZeroOne 的最终输出控制层之一使用，帮助把：
- 线索
- 观察判断
- 可执行结论
明确分层表达。

这样可以减少：
- 轻重不分
- 强弱不清
- 结论表达过满
并提升分析输出的交易可用性。
