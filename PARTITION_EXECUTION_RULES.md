# PARTITION_EXECUTION_RULES.md

更新时间：2026-03-31

## 目的
把“同仓库分区”从结构层，落实到执行层，避免龙皓晨、王林与 shared 之间出现读写混乱。

## 总原则
1. shared 只放共用资产。
2. 龙皓晨与王林除共享层外，其他沉淀必须分区。
3. 王林不得写入龙皓晨专属区；龙皓晨不得把王林专属沉淀混写到自己区。
4. 若内容同时适用于两者，优先写入 shared，而不是任意一方专属目录。
5. 任何新增文件，优先从一开始就写进正确分区，不再混放。

## 当前系统入口说明
- 顶层 `MEMORY.md`、`REBOOT_RULES.md`、`WORKFLOW_DAILY_RADAR_V3.md` 目前仍作为主系统运行入口，默认对应龙皓晨主线。
- 这样做是为了不破坏现有 cron、恢复链路和工作流引用。
- 未来如需进一步彻底迁移，再分批把顶层龙皓晨资产迁入 `agents/longhaochen/`，但当前阶段不强制搬迁旧文件。

## 读写规则

### 一、shared/
#### 读取
- 龙皓晨：可读
- 王林：可读

#### 写入
仅当内容满足以下条件时才写入：
- 两个角色都要复用
- 属于系统级规则 / workflow / 模板 / 模块 / 公共记忆

#### 示例
- 共用 workflow
- 共用模板
- 共用模块
- 共用执行治理规则

### 二、agents/longhaochen/
#### 读取
- 龙皓晨：优先读取
- 王林：默认不读取，除非景森明确要求协同引用

#### 写入
- 只写龙皓晨的主报告沉淀、总裁决、跨市场判断、组长规则、总复盘

### 三、agents/wanglin/
#### 读取
- 王林：优先读取
- 龙皓晨：可在总调度、检查执行、汇总复盘时读取

#### 写入
- 只写王林的加密执行日志、执行策略、执行规则、加密复盘、执行模板

## 王林默认执行边界
### 读取顺序
1. `shared/`
2. `agents/wanglin/`

### 写入顺序
1. 优先写入 `agents/wanglin/`
2. 若内容明确属于公共资产，再写入 `shared/`
3. 不得默认写入龙皓晨专属区

## 龙皓晨默认执行边界
### 读取顺序
1. 顶层主系统入口文件
2. `shared/`
3. `agents/longhaochen/`
4. 如需调度/复盘王林执行线，可读取 `agents/wanglin/`

### 写入顺序
1. 主系统与总控层内容：顶层 + `agents/longhaochen/`
2. 公共内容：`shared/`
3. 不把王林专属执行沉淀混写到自己专属区

## 恢复规则
### 恢复龙皓晨
- 读取：顶层系统入口 + `shared/` + `agents/longhaochen/`

### 恢复王林
- 读取：`shared/` + `agents/wanglin/`

### 恢复 shared
- 只恢复公共资产，不混入角色专属沉淀

## Skill 双层目录治理补充（2026-03-31）
### 一、目录分工
#### 1. `workspace/skills/`
- 定位：共享主层。
- 作用：龙皓晨、王林都可调用的通用、稳定、可复用 skill。

#### 2. `workspace/.agents/skills/`
- 定位：王林执行专属层。
- 作用：王林当前执行线专用、试运行、定制化、未完成共享化的 skill。

### 二、读取规则
#### 王林
1. 优先读取 `workspace/.agents/skills/`
2. 再读取 `workspace/skills/`

#### 龙皓晨
1. 优先读取 `workspace/skills/`
2. 如需核对王林执行能力，再读取 `workspace/.agents/skills/`

### 三、写入规则
#### 写入 `workspace/skills/` 的条件
- 两个实例都要长期复用；或
- 已验证稳定；或
- 已明确属于正式生产层。

#### 写入 `workspace/.agents/skills/` 的条件
- 只服务王林执行线；或
- 仍在测试期；或
- 尚未确认长期保留；或
- 带明显执行专属属性。

### 四、升格规则
从 `.agents/skills/` 升格到 `skills/`，必须同时满足：
1. 已实际可用
2. 至少完成一轮实战验证
3. 不只是王林个人临时需求
4. 已明确会长期复用

### 五、重复 Skill 去重规则
若两个目录出现同名 skill：
1. 先标记
2. 明确主版本
3. 未确认前不立刻硬删
4. 确认后只保留一份主版本，另一份删除或归档

### 六、`hyperliquid` 双份去重策略
- 正式主版本：`workspace/skills/hyperliquid`
- `.agents/skills/hyperliquid` 仅作为临时比对对象，不长期保留
- 若专属层无额外定制改动：删除 `.agents/skills/hyperliquid`
- 若专属层存在有效定制改动：先合并进共享层，再删除专属层重复版本

## 当前执行策略
- 旧文件不强制大迁移，避免影响运行。
- 新增内容从现在开始严格按分区写入。
- 若后续出现混放，再做定向迁移与收口。
- 新装 skill 默认先进入 `workspace/.agents/skills/`，验证稳定后再决定是否升格进 `workspace/skills/`。
