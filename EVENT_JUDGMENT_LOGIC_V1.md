# 事件判断分析逻辑 V1

更新时间：2026-04-03

## 一、目标
事件判断不是“补充消息”，而是主报告分析的起点。
它要回答：
1. 今天有哪些重要事件？
2. 哪些事件真的重要？
3. 这些事件会影响哪一层？
   - 宏观
   - 流动性
   - 风险偏好
   - 资金流
   - 某条链 / 某赛道 / 某市场
4. 市场当前在怎么定价这些事件？

## 二、事件判断的正确顺序
### 第一步：先列事件
先把当天事件按类别列出来，不急着分析。

分 4 类：
1. 宏观 / 政策类
- 美联储
- CPI / 非农
- 利率
- 监管政策
- 关键官员讲话

2. 地缘 / 政治类
- 战争
- 制裁
- 谈判
- 选举
- 特朗普 / 核心政客讲话

3. 市场结构类
- ETF
- 交易所
- 稳定币
- 清算
- 大额爆仓
- 大额资金流入流出

4. 生态 / 协议类
- 被盗
- 宕机
- 脱锚
- 安全事故
- 协议异常
- 链上基础设施问题

### 第二步：筛选重大事项
不是所有事件都写进主报告。要筛一遍，判断是否满足至少一条：
1. 会影响风险偏好
2. 会影响预期
3. 会影响资金流向
4. 会影响某个生态 / 赛道估值
5. 会引发跨市场联动
6. 市场已经明显开始定价

满足其中一条，就进入主报告事件判断。

### 第三步：判断事件层级
每个事件都要判断它属于哪一层：

A. 全市场级
- 会影响宏观 / 流动性 / 风险偏好 / 多市场同步波动
- 例如：美联储会议、特朗普重大讲话、战争升级

B. 市场级
- 会影响某一个大市场：加密 / 美股 / 港股 / 商品 / 预测市场
- 例如：ETF 审批、监管事件、稳定币风险

C. 生态 / 赛道级
- 会影响某条链 / 某个协议赛道 / 某类资产风险偏好
- 例如：Drift 被盗、Solana 生态协议异常、某 DeFi 协议脱锚

### 第四步：判断事件作用方向
每个重大事件都要明确回答：
1. 利多 / 利空 / 中性？
2. 影响短期还是中期？
3. 影响是局部还是扩散？
4. 是已经落地，还是市场在交易预期？

### 第五步：判断市场是否在定价
事件不能只看新闻，要看市场是否相信。
至少看：
1. 价格是否反应
2. 资金是否反应
3. 预测市场是否反应
4. 相关标的强弱是否变化
5. 是否出现扩散 / 跟随 / 风险切换

如果市场没反应：
- 可能不重要
- 可能还没到定价时点
- 可能只是噪音

## 三、事件判断最终输出格式
每个纳入主报告的事件，最终都应压缩成 4 句话：
1. 事件是什么
2. 它影响哪一层（宏观级 / 市场级 / 生态级）
3. 它会怎么影响预期 / 风险偏好 / 资金流
4. 市场有没有开始定价

## 四、特别要强制检查的两类事件
### 1. 预期型重大事项
必须提前看，不是发生后再补：
- 特朗普讲话
- 美联储会议
- CPI / 非农
- 政策发布
- 监管听证
- 选举节点

关键不是复述内容，而是提前判断：
- 市场在押什么
- 哪个方向最脆弱
- 哪些市场最敏感

### 2. 协议 / 生态级风险事项
必须强制纳入检查：
- 被盗
- 清算
- 宕机
- 脱锚
- 协议异常
- 链上基础设施故障

关键不是“是不是全市场主线”，而是判断：
- 会不会伤害该生态风险偏好
- 会不会影响赛道资金流
- 会不会扩散到相关资产

## 五、避免漏项的最小检查
写主报告前先做三问：
1. 今天有哪些重要事件？
2. 哪些事件真的会影响预期 / 风险偏好 / 资金流？
3. 哪些事件市场已经在定价？

这三问没答完，主报告不能开写。

## 六、一句话结论
事件判断要先列事件、再筛重大、再分层级、再判方向、再看市场是否定价，最后才能进入主报告分析。
