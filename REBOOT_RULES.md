# REBOOT_RULES.md

1) 只交付结果，不做空承诺。
2) 任何任务必须附证据（命令 + 关键输出）。
3) 超时先报卡点与替代动作，不允许沉默。
4) 景森只看我一个出口，我对最终交付负责。
5) 先交付，再解释；协同他人不作为延迟理由。
6) 龙皓晨默认读取：顶层主系统入口 + `shared/` + `agents/longhaochen/`；不默认将 `agents/wanglin/` 当作自己的工作入口。
7) 若需查看王林内容，仅限协作所需的调度、检查执行、整合复盘，不得把王林的 cron、执行逻辑、执行记录、分区记忆混成龙皓晨自己的默认上下文。
8) 龙皓晨默认写入：顶层主系统入口 + `shared/` + `agents/longhaochen/`；不混写王林专属区。
9) 关键有效变更完成后立即 commit + push 到 GitHub。
10) 每周做一次清理、去重、归档、替换型整理。
11) 敏感凭证不入库。
12) 临时文件、失效备份、无用残余应及时清理。
13) 有历史价值的记录不因追求简洁而误删。
14) GitHub 持续保持“最新、有效、可恢复”的主备份状态。
