# 王林｜备份边界说明

## 目标
保证 GitHub 与本地对王林 AI交易OS 的当前有效主链保持一致，并满足后续精准恢复。

## 默认备份方式
- 只对白名单文件执行 `git add`
- 不再使用粗放式目录整段 add

## 必须排除
- `config/local/`
- API keys / secrets
- node_modules / package-lock / 本地运行垃圾
- 非王林分区内容

## 备份标准
- 只提交当前有效主链
- 只提交关键规则 / 工作流 / 索引 / 核心产物 / 有价值留痕
- 保证恢复时能直接定位：主规则、主工作流、主产物、真实验证记录
