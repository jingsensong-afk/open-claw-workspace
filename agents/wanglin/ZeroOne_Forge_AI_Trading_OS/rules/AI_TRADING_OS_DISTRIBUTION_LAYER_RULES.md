# 王林｜AI交易OS 第五层内容分发层规则

## 目标
把 AI交易OS 各层输出整理成适合外部查看和后续联动的标准内容。

## 当前阶段优先级
1. Telegram
2. 币安广场（后续验证）
3. X（后续验证）
4. 其他渠道

## 当前分发层职责
1. 汇总信息流层、候选池层、思考层、执行层的结果
2. 生成标准化简洁输出
3. 优先满足 Telegram 自动输出
4. 为后续币安广场 / X 接入预留格式


## 币安广场样稿风格补充（2026-04-26新增）
- 币安广场默认采用一句话、简洁、重点、说而不说的风格。
- 每条贴文默认先挂币种小组件对应的币种语义，文字里直接用 `$DOGE`、`$BTC` 这种形式自然接内容，不再写 `DOGE long:`、`BTC short:` 这种纯文本前缀。
- 若系统存在真实开单，则优先采用“币种 + 一句贴币判断 + 交易/开单组件”的结构，而不是只发文字。
- 内容要兼具交易员判断与营销钩子：既点到为什么值得看/为什么敢开这个单，又要保留悬念，不把逻辑讲满。
- 默认不发系统统计数字，不直接堆 execution 清单，不写成报告体。
