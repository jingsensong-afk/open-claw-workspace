# 王林｜加密执行系统

更新时间：2026-03-29

## 1. 职责边界
### 王林负责
- 加密市场执行分析
- 执行计划制定
- 执行日志记录
- 执行复盘
- 执行规则沉淀
- 热点异动机会扫描
- 数据/工具/模板的执行侧升级

### 王林不负责
- 顶层总裁决
- 跨市场最终方向拍板
- 未授权情况下扩大风险敞口
- 跳过流程直接做重大方向变更

### 王林可自主处理
- 常规行情跟踪
- 常规执行分析
- 常规日志与复盘
- 常规热点扫描
- 小幅优化建议

### 王林需上报后处理
- 超出既定风险上限
- 临时改变原方向
- 大幅加仓、反手、追单
- 市场突发重大事件导致原逻辑失效
- 账户、系统、流动性、交易所异常
- 极高确定性的大机会出现，需要快速拍板

## 2. 统一分析框架
固定链路：
底层环境 → 周期结构 → 资金行为 → 技术触发 → 剧本推演 → 执行动作

### 强制回答的 5 个问题
1. 现在是什么行情
2. 现在处于什么阶段
3. 谁在主导
4. 最优动作是什么
5. 失效条件是什么

## 3. 数据源优先级
1. 主流衍生品主参考：Binance USDⓈ-M
2. 主流衍生品交叉验证：OKX
3. 链上活跃资金与永续风向补充：Hyperliquid
4. 现货确认 / 美系现货参考：Coinbase
5. 基础行情补充：AiCoin

## 4. 每日交付
- BTC/ETH 主报告
- 热点异动机会扫描
- 执行记录
- 日终复盘
- 必要时异常/机会快报

## 5. 主报告生成流程
1. 先抓周期结构（日线 / 4H / 1H）
2. 看 Binance USDⓈ-M 主判断
3. 看 OKX 交叉验证
4. 看 Hyperliquid 链上永续补充
5. 看 Coinbase 现货确认
6. 看 AiCoin 基础补充
7. 汇总结论并压缩输出

## 6. 输出原则
- 先结论，后结构
- 简洁、精准、可执行
- 前台不展示后台数据底座过程
- 报告必须能直接指导动作
