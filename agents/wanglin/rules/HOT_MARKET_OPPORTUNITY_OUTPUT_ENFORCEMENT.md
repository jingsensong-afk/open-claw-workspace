# 王林｜热点异动报告输出硬约束

更新时间：2026-04-18

## 目标
确保晚间直播前精简异动/预热报告既能给出可执行主推机会，也不会漏掉群内 / 市场已形成关注度的事件驱动或叙事驱动标的；同时必须足够精简，能直接承接直播。

## 强制输出结构
每次晚间直播前精简异动/预热报告必须默认拆成两层：

### A类：今晚主推异动标的（可直接执行）
- 用于输出可直接执行、可直播承接的异动机会
- 必须能落到：是否值得做、方向判断、进场区间、止损位、止盈位、仓位建议、主/次/反向剧本、最优动作、关键位、失效条件

### B类：热点异动观察名单
- 用于覆盖群内 / 市场高关注但暂不主推的热点币
- 即使不纳入主推，也不得在正文完全缺席
- 每个标的至少写清：
  - 是否纳入主推：是 / 否
  - 未纳入主推原因：
  - 当前异动核心：
  - 当前阶段：
  - 观察结论：做 / 不做 / 观望
  - 若后续转强，升级为主推需满足：
  - 可参考进场点：
  - 失效点：
  - 当前主要风险：

## 强制覆盖规则
1. 若群内已明确点名、讨论或持续关注某热点币，即使不纳入主推，也必须进入 B类观察名单。
2. 若市场出现明显事件驱动 / 叙事驱动热点币，即使其不属于主流高流动性池，也必须至少纳入观察名单。
3. 不得再出现“群里都在看，但正文完全没提”的情况。

## 第6/7维表达约束
1. OI / Funding 已有固定数据源，不得再笼统写成“无来源缺口”或泛化“未完整补齐”。
2. 若异动报告需要引用第6/7维验证状态，必须区分：
   - 未按 checklist 完成核查；
   - 已核查但未写入前台；
   - Binance + OKX 同时异常。
3. 若真正未完整补齐的是第8/9维，必须只写第8/9维自己的真实缺口，不得打包混进第6/7维。

## 降级表达规则
1. 若关键证据不足，必须直接写清，不得硬拔高结论。
2. 若判断明显依赖主报告缺失维度，必须写明“仍需等待更多确认”。
3. 若标的更偏情绪驱动、流动性差、滑点大、持续性弱，可纳入 B类观察名单，但不得硬写成 A类主推。

## 禁止事项
1. 禁止只写主流可执行标的，漏掉群内高关注热点币。
2. 禁止未纳入主推却不给原因。
3. 禁止把执行未落地表述成数据源缺失。
4. 禁止对事件/叙事币只因涨幅大就直接拔高为高确定性主推。

## 直播前精简表达规则
1. 默认控制在“短、直接、可开播前快速看完”的风格，避免写成独立长报告。
2. A类主推机会若不给出明确进场区间、止损位、止盈位，不算合格。
3. 若点位只能给区间，不得只喊方向不报位置。
4. 若波动过快导致无法给精确单点，至少给可执行区间 + 失效点。
5. 必须明确写出今晚直播优先关注的主推标的或方向。

## 合格判定
一份晚间直播前精简异动/预热报告只有在以下条件同时满足时才算合格：
1. 使用标准模板输出；
2. A类主推机会可直接指导动作，且已明确进场区间、止损位、止盈位；
3. B类已覆盖群内 / 市场高关注热点币；
4. 未纳入主推的热点币已明确说明原因；
5. 关键证据边界与主报告当前证据边界保持一致；
6. 未再把第6/7维笼统写成“缺失”；
7. 整体表达足够精简，可直接承接直播。
