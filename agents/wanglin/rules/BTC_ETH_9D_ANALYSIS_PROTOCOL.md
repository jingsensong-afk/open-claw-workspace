# 王林｜BTC/ETH 主报告 9维后台分析协议

更新时间：2026-04-07

## 目标
在输出 BTC/ETH 主报告前，必须先完成 9 个核心维度的后台分析，再压缩成前台可执行报告。
前台可以简洁，但后台不得偷工减料。

## 强制 9 维
### 1. 价格结构（高低点）
- 日线 / 4H / 1H 的高低点结构
- 判断 higher high / higher low 或 lower high / lower low
- 判断当前是否处于趋势延续、震荡、反转尝试

### 2. 成交量
- 放量突破 / 缩量回踩 / 放量滞涨 / 放量下跌
- 量价是否共振
- 当前量能是否支持趋势延续

### 3. EMA（趋势成本）
- 至少检查主趋势成本带
- 价格是否站上 / 跌破关键 EMA
- EMA 排列是否顺向发散或收敛拐头

### 4. MACD（动能）
- 金叉 / 死叉
- 柱体扩张 / 收缩
- 是否出现背离风险

### 5. RSI（节奏）
- 强弱区间
- 是否超买 / 超卖
- 是否出现顶背离 / 底背离

### 6. OI（杠杆）
- 未平仓变化方向
- 配合价格与成交量判断是增仓推动、减仓上涨、增仓下跌还是挤仓
- 若有条件，补多空持仓比

### 7. Funding（拥挤）
- 多头是否过热
- 空头是否过热
- 是否具备 squeeze 条件

### 8. 清算热图（流动性）
- 上下方清算密集区
- 未平仓集中区
- 更可能先被扫的是哪一侧
- 当前稳定优先主方案：CoinGlass API
- 过渡来源：AiCoin（liquidation_map / liquidation_history / estimated_liquidation）
- 当前辅助来源：Hyperliquid 清算 / 大额仓位相关公开数据
- 若无法稳定拿到热图区与集中区，不得把本维度写成“已完整分析”

### 9. 宏观流动性
- 美债收益率
- DXY
- 美股指数
- 期货贵金属走势
- 资金流数据
- 判断外部风险偏好是否支持当前 crypto 方向
- 本维度拆成两层：
  1. 风险偏好层：美债 / DXY / 美股 / 贵金属 / 原油（稳定优先主来源：Trading Economics）
  2. crypto资金流层：稳定币 / BTC / ETH / ETF资金流（稳定币优先：DefiLlama Stablecoins；BTC ETF优先：BTCETFData）
- 若只完成第一层、未完成第二层，必须明确写为“宏观流动性部分完成，不是完整完成”

#### 第9维｜crypto资金流层固定后台字段
1. 稳定币总市值变化
2. 稳定币净流入 / 净流出方向
3. BTC ETF 当日净流方向
4. BTC ETF 持仓变化
5. 若暂时拿不到 ETH ETF 数据，必须显式标注“ETH ETF流暂缺”

#### 第9维｜crypto资金流层默认来源
- 稳定币：DefiLlama Stablecoins
  - 默认优先端点：
    - `https://api.llama.fi/stablecoins`
    - `https://api.llama.fi/stablecoincharts/all`
    - `https://api.llama.fi/stablecoinchains`
- BTC ETF：BTCETFData
  - 当前默认采用网页稳定抓取字段：ticker / holdings / change / as-of date
  - 若后续确认 JSON 入口，切换为 JSON 优先
- ETH ETF：后续补链

#### 第9维｜crypto资金流层输出原则
- 不要求前台逐项报数，但后台必须逐项检查
- 前台至少要压缩成：
  - 资金流偏风险偏好 / 偏收缩 / 中性
  - crypto增量资金是否支持当前方向
- 若仅有稳定币或仅有BTC ETF其中一项，不得写成“资金流完整支持”，只能写部分支持或证据不足

## 输出规则
1. 必须先完成后台 9 维，再生成前台主报告。
2. 前台主报告仍严格使用 `agents/wanglin/templates/BTC_ETH_MAIN_REPORT_TEMPLATE.md`。
3. 前台不必逐条展示 9 维，但结论必须来自 9 维综合判断。
4. 若关键维度缺失，必须主动降级表达，不得高置信拍板。

## 关键维度缺失时的降级规则
### 可降级但仍可给结论
- 缺 1-2 个非核心维度时，可输出结论，但必须明确“结论置信度下降”。

### 必须降级为谨慎判断
若以下任一项缺失，必须显式降级：
- OI
- Funding
- 清算热图
- 宏观流动性

### 不得高置信拍板的情形
若以下两类同时缺失，不得给高置信方向判断：
- 杠杆/拥挤/流动性链（OI、Funding、清算热图）
- 外部流动性链（宏观流动性）

## 数据源优先级建议
1. 主流衍生品主参考：Binance USDⓈ-M
2. 主流衍生品交叉验证：OKX
3. 链上活跃资金与永续风向补充：Hyperliquid
4. 现货确认 / 美系现货参考：Coinbase
5. 基础行情补充：AiCoin
6. 宏观数据源：DXY / 美债 / 美股 / 贵金属 / 资金流

## 第6维 / 第7维 固定取数顺序
### 6. OI（杠杆）
1. 先看 Binance USDⓈ-M
2. 再看 OKX 是否同向 / 背离
3. 再看 Hyperliquid 是否强化或削弱该判断
4. 若 Binance 与 OKX 同向，则可提高该维度置信度
5. 若 Binance 与 OKX 背离，则前台必须降级表达，不得写成高置信单边判断

### 7. Funding（拥挤）
1. 先看 Binance USDⓈ-M
2. 再看 OKX 是否同向 / 背离
3. 再看 Hyperliquid 是否强化或削弱该判断
4. 若 Binance 与 OKX 同向，则可提高该维度置信度
5. 若 Binance 与 OKX 背离，则前台必须降级表达，不得写成高置信单边判断

## 验收标准
一份 BTC/ETH 主报告只有在以下条件同时满足时，才可视为合格：
1. 后台 9 维已逐项检查；
2. 关键缺口已显式说明；
3. 前台报告能落到动作、关键位、剧本切换条件、失效条件。
