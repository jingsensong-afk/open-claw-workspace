# 每日投研机会雷达工作流 V3（实战裁决增强版）

更新时间：2026-04-01

## 一、目标
升级为“**底层判断 + 事件变量 + 中层推演 + 各市场变化判断**”的一次性每日主报告体系。

当前版本不再追求组长直接给出各市场具体执行口径，而是由组长负责：
- 洞悉周期 / 宏观 / 流动性 / 资金流
- 纳入重要时事 / 热点 / 重大事件
- 判断这些变量如何压制并改变各市场结构
- 输出跨市场变化结论与优先级

该工作流用于：
- 快速识别当日主导变量与核心交易主线
- 把慢变量与事件变量统一到同一条分析链
- 输出各市场变化判断，而非替执行员给出具体执行策略
- 保留完整证据链与留痕能力
- 在保证可审计性的前提下，提高实战可读性与决策速度

## 二、V3 升级原则
1. **先裁决，再展开**：先给“今日主导变量 / 今日一句话结论”，再展开证据链。
2. **主线优先于铺陈**：优先回答“今天市场在交易什么、该怎么做”，避免模块齐全但主线不清。
3. **V2 模块不丢失**：新闻、舆情、结构、预测市场、链上、情绪、跨市场、风险提示等模块仍需覆盖。
4. **显式标注降级项**：凡数据源不可用、证据不足、模块缺口，必须明确写“冲突点 / 采用口径 / 待验证项”。
5. **结论必须落到变化判断**：所有判断都应尽量落到“当前变化、优先级、风险点、该关注什么 / 不该误判什么”。
6. **优先级必须前置**：主导变量、最强信号、次强信号、被降级信号必须进入主输出，而不是埋在尾部。

## 三、主输出结构（11:00）

### A. 事件判断前置（硬执行步骤，未完成不得进入正文）
0. **先完成事件判断，再开写主报告**
   - 先列出当日重要时事 / 热点 / 重大事件。
   - 再筛选哪些事件会影响预期 / 风险偏好 / 资金流 / 生态或赛道估值。
   - 再判断事件层级：全市场级 / 市场级 / 生态级。
   - 再判断市场是否已经开始定价。
   - 若以上步骤未完成，不得进入主报告正文。
1. **今日一句话结论**
   - 直接回答：今天市场主线是什么，当前该偏进攻、防守还是观望。
2. **主导变量**
   - 如：地缘风险 / 政策边际变化 / 风险偏好修复 / 杠杆出清 / 事件扩散。
3. **当前交易态度**
   - 防守优先 / 轻偏多 / 轻偏空 / 观望 / 只做反抽 / 等待确认。
4. **主交易框架**
   - 主流趋势 / 事件驱动 / 超跌反抽 / 宏观对冲 / 概率交易。

### B. 证据层（按 V2 保留，但表达更聚焦）
5. 重大事件 / 热点事件
6. 叙事扩散情况
7. 新闻 / 宏观触发响应
8. 市场是否相信（价格 / OI / funding / 量能 / 预测市场 / 钱包行为）
9. 资金流向判断
10. 跨市场联动判断
11. 链上 / 鲸鱼 / 现货资金验证
12. 清算热图 / 流动性热区 / 短周期拐点预警
13. OI + 多空持仓比完整验证
14. Funding / 情绪 / Fear & Greed 统一面板
15. 产业瓶颈 / 价格错配（如适用）
16. Token / 地址 / 热门信息一体化验证

### C. 变化判断层
17. 加密市场变化判断
18. 美股市场变化判断
19. 港股市场变化判断
20. 大宗商品变化判断
21. 预测市场变化判断
22. 信号优先级汇总
   - 当前主导变量
   - 最强信号
   - 次强信号
   - 被降级信号
23. 当前置信度
24. 风险提示

## 四、输出频率
- 每日只输出一次主报告
- 不再固定输出 15:30 增量补丁
- 如遇重大突发事件，再单独输出异常提示

## 五、调用要求（继承 V2，增加显式裁决）
1. 主场景必须先路由（交易机会筛选 / 热点事件投研 / 实盘异动诊断）。
2. 必须优先识别“主导变量”，避免平均用力。
3. 事件判断必须前置，至少完成以下四步：
   - 列出当日重要时事 / 热点 / 重大事件
   - 筛选哪些事件会影响预期 / 风险偏好 / 资金流 / 生态或赛道估值
   - 判断事件层级：全市场级 / 市场级 / 生态级
   - 判断市场是否已经开始定价
4. 若某模块数据缺失，不允许静默跳过，必须写明：
   - 冲突点
   - 当前采用口径
   - 待验证项
5. 若出现明显跨市场主线（如地缘、油金、政策、风险偏好切换），跨市场联动必须升权。
6. 若高弹性标的弱于主流币/主流资产，必须明确标记为“反抽框架”而非“趋势框架”。
7. 若事件判断未完成或重大事项漏判，视为主报告质量不合格。

## 六、表达规范
1. 先说结论，再说为什么。
2. 尽量避免“信息很多但结论模糊”。
3. 优先使用：
   - 主导变量
   - 资金流向
   - 市场是否相信
   - 当前该怎么做
4. 每份主报告必须至少给出一个“一句话执行结论”。
5. 允许实战语气更强，但不能牺牲证据链。

## 七、V2 与 V3 的关系
- V2：标准化、模块完整、便于留痕与审计。
- V3：在保留 V2 模块完整性的前提下，增强交易员裁决表达、主导变量识别与执行落地性。

默认执行原则：
- **每日 11:00 按 V3 输出主报告**。
- 预测市场分析与每日复盘分别按独立机制执行。
- 若需审计或模块复核，回退查看历史版本与归档文件。

## 八、当前结论
V3 是 ZeroOne Venture 当前阶段更适合实战的每日主报告版本：
- 先做底层判断
- 再纳入事件变量
- 再做中层推演
- 最终落到跨市场变化判断

即：**先抓本质，再看变化；组长给结论，各市场执行员给执行。**
