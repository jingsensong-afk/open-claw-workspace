# 每日投研机会雷达工作流 V3（实战裁决增强版）

更新时间：2026-03-27

## 一、目标
在 V2 的完整证据链与模块化结构基础上，升级为“**交易员裁决优先 + 标准化留痕并行**”的主报告体系。

V3 不是削弱 V2，而是把 V2 的完整框架与实战读盘、主导变量识别、跨市场裁决表达融合，形成更适合 ZeroOne Venture 当前协作场景的主报告版本。

该工作流用于：
- 快速识别当日主导变量与核心交易主线
- 保留完整证据链与模块级验证
- 输出可直接执行的主交易标的 / 高弹性标的 / 观察标的
- 强化触发条件 / 执行动作 / 失效条件三元组
- 在保证可审计性的前提下，提高实战可读性与决策速度

## 二、V3 升级原则
1. **先裁决，再展开**：先给“今日主导变量 / 今日一句话结论”，再展开证据链。
2. **主线优先于铺陈**：优先回答“今天市场在交易什么、该怎么做”，避免模块齐全但主线不清。
3. **V2 模块不丢失**：新闻、舆情、结构、预测市场、链上、情绪、跨市场、风险提示等模块仍需覆盖。
4. **显式标注降级项**：凡数据源不可用、证据不足、模块缺口，必须明确写“冲突点 / 采用口径 / 待验证项”。
5. **结论必须落到动作**：所有判断都应尽量落到交易框架（做 / 不做 / 只观察 / 轻仓试错 / 等待确认）。
6. **优先级必须前置**：主导变量、最强信号、次强信号、被降级信号必须进入主输出，而不是埋在尾部。

## 三、主输出结构（11:00）

### A. 裁决层（前置，必须先写）
1. **今日一句话结论**
   - 直接回答：今天市场主线是什么，当前该偏进攻、防守还是观望。
2. **主导变量**
   - 如：地缘风险 / 政策边际变化 / 风险偏好修复 / 杠杆出清 / 事件扩散。
3. **当前交易态度**
   - 防守优先 / 轻偏多 / 轻偏空 / 观望 / 只做反抽 / 等待确认。
4. **主交易框架**
   - 主流趋势 / 事件驱动 / 超跌反抽 / 宏观对冲 / 概率交易。

### B. 证据层（按 V2 保留，但表达更聚焦）
5. 重大事件 / 热点事件
6. 叙事扩散情况
7. 新闻 / 宏观触发响应
8. 市场是否相信（价格 / OI / funding / 量能 / 预测市场 / 钱包行为）
9. 资金流向判断
10. 跨市场联动判断
11. 链上 / 鲸鱼 / 现货资金验证
12. 清算热图 / 流动性热区 / 短周期拐点预警
13. OI + 多空持仓比完整验证
14. Funding / 情绪 / Fear & Greed 统一面板
15. 产业瓶颈 / 价格错配（如适用）
16. Token / 地址 / 热门信息一体化验证

### C. 执行层（必须可交易）
17. 加密市场：主交易标的 / 高弹性标的 / 观察标的
18. 美股市场：主交易标的 / 高弹性标的 / 观察标的
19. 港股市场：主交易标的 / 高弹性标的 / 观察标的
20. 大宗商品：主交易标的 / 观察标的
21. 预测市场：主交易标的 / 观察标的
22. **执行三元组**
   - 触发条件
   - 执行动作
   - 失效条件
23. 信号优先级汇总
   - 当前主导变量
   - 最强信号
   - 次强信号
   - 被降级信号
24. 当前置信度
25. 时间维度分层（短线 / 日内 / 波段 / 中线）
26. 风险提示

## 四、15:30 增量补丁结构
15:30 不重复 11:00 全量结构，重点回答“**11:00 之后什么变了**”：
1. 今日 11:00 后新增重大信息 / 热点
2. 主导变量是否变化
3. 哪个信号增强 / 哪个信号减弱 / 哪个信号失效
4. 价格 / OI / funding / 多空比 / 量能 / 预测市场 / 链上资金是否出现关键增量
5. 是否需要更新：
   - 主交易标的
   - 高弹性标的
   - 观察标的
   - 触发条件
   - 失效条件
6. 当前一句话更新结论

## 五、调用要求（继承 V2，增加显式裁决）
1. 主场景必须先路由（交易机会筛选 / 热点事件投研 / 实盘异动诊断）。
2. 必须优先识别“主导变量”，避免平均用力。
3. 若某模块数据缺失，不允许静默跳过，必须写明：
   - 冲突点
   - 当前采用口径
   - 待验证项
4. 若出现明显跨市场主线（如地缘、油金、政策、风险偏好切换），跨市场联动必须升权。
5. 若高弹性标的弱于主流币/主流资产，必须明确标记为“反抽框架”而非“趋势框架”。
6. 若结论无法落到标的与条件，视为主报告质量不合格。

## 六、表达规范
1. 先说结论，再说为什么。
2. 尽量避免“信息很多但结论模糊”。
3. 优先使用：
   - 主导变量
   - 资金流向
   - 市场是否相信
   - 当前该怎么做
4. 每份主报告必须至少给出一个“一句话执行结论”。
5. 允许实战语气更强，但不能牺牲证据链。

## 七、V2 与 V3 的关系
- V2：标准化、模块完整、便于留痕与审计。
- V3：在保留 V2 模块完整性的前提下，增强交易员裁决表达、主导变量识别与执行落地性。

默认执行原则：
- **11:00 / 15:30 以后优先按 V3 输出**。
- 若需审计或模块复核，回退查看 V2 结构要求。

## 八、当前结论
V3 是 ZeroOne Venture 当前阶段更适合实战的每日雷达版本：
- 先做交易裁决
- 再补完整证据
- 最终落到可执行标的与条件

即：**标准化骨架不丢，交易员锋利表达前置。**
